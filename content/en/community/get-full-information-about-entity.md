The completeness of the entity information depends of parameter on request header.

Usual request looks like

```json
OData-Version:4.0
OData-MaxVersion:4.0
Content-Type:application/json;odata.metadata=minimal
Accept:application/json;odata.metadata=minimal
Accept-Charset:UTF-8
Authorization:Bearer {{token}}
Host:{{base_url}}
```

When we are trying describe requirement for destination system our requests for response. 

`Accept:application/json;odata.metadata=minimal` we can accept only `json` response with `minimal` information represent inside

<br/>

`@odata.id` - direct link for current record (in case we have got few records for request it can be useful) and allow to undestand key fields in brackets `https://{{base_url}}/data/SalesOrderHeadersV2(dataAreaId='usmf',SalesOrderNumber='000002')`

Additional we can get rellation of this entity with different entityes. For example `"SalesOrderLines@odata.navigationLink": "https://{{base_url}}/data/SalesOrderHeadersV2(dataAreaId='usmf',SalesOrderNumber='000002')/SalesOrderLines"` allow to get all headers and lines in one query.
